## A Evolução do CSS

O CSS deu um salto enorme nos últimos anos. O que antes exigia JavaScript ou frameworks hoje é feito com poucas linhas de CSS puro.

## CSS Grid

A forma mais poderosa de criar layouts bidimensionais:

```css
.dashboard {
  display: grid;
  grid-template-columns: 250px 1fr 300px;
  grid-template-rows: auto 1fr auto;
  grid-template-areas:
    "header  header  header"
    "sidebar content aside"
    "footer  footer  footer";
  gap: 1rem;
  min-height: 100vh;
}

.header  { grid-area: header; }
.sidebar { grid-area: sidebar; }
.content { grid-area: content; }
.aside   { grid-area: aside; }
.footer  { grid-area: footer; }

/* Responsivo com auto-fit */
.galeria {
  display: grid;
  grid-template-columns: repeat(auto-fit, minmax(250px, 1fr));
  gap: 1rem;
}
```

## Container Queries

Diferente de Media Queries (que olham o viewport), Container Queries olham o tamanho do **container pai**:

```css
/* Definir um container de query */
.card-wrapper {
  container-type: inline-size;
  container-name: card;
}

/* Estilos baseados na largura do container */
@container card (min-width: 400px) {
  .card {
    display: grid;
    grid-template-columns: 200px 1fr;
    gap: 1rem;
  }
  .card img {
    width: 100%;
    height: 100%;
    object-fit: cover;
  }
}

@container card (max-width: 399px) {
  .card {
    display: flex;
    flex-direction: column;
  }
  .card img {
    width: 100%;
    aspect-ratio: 16 / 9;
  }
}
```

## A Pseudo-classe `:has()`

O "parent selector" que o CSS sempre precisou:

```css
/* Estilizar um card que contém uma imagem */
.card:has(img) {
  padding: 0;
  overflow: hidden;
}

/* Estilizar formulário que tem erro */
form:has(.campo-invalido) {
  border: 2px solid red;
  padding: 1rem;
}

/* Grid responsivo sem media queries */
.container:has(.card:nth-child(5)) {
  grid-template-columns: repeat(3, 1fr);
}

.container:has(.card:nth-child(3)) {
  grid-template-columns: repeat(2, 1fr);
}
```

## CSS Layers (`@layer`)

Controle explícito da especificidade e ordem de cascata:

```css
@layer reset, base, componentes, utilitarios;

@layer reset {
  *, *::before, *::after {
    margin: 0;
    padding: 0;
    box-sizing: border-box;
  }
}

@layer base {
  body {
    font-family: sans-serif;
    line-height: 1.6;
    color: #1a1a1a;
  }
}

@layer componentes {
  .btn {
    padding: 0.5rem 1rem;
    border-radius: 0.5rem;
    font-weight: 600;
  }
  .btn-primary {
    background: #3b82f6;
    color: white;
  }
}

@layer utilitarios {
  .sr-only { /* acessibilidade */ }
  .truncate { overflow: hidden; text-overflow: ellipsis; white-space: nowrap; }
}
```

## Animações CSS Performáticas

Prefira `transform` e `opacity` — eles não disparam layout nem paint:

```css
/* ❌ Ruim: anima propriedades que disparam layout */
.card-ruim {
  transition: width 0.3s, height 0.3s, left 0.3s, top 0.3s;
}

/* ✅ Bom: anima apenas transform e opacity */
.card-bom {
  transition: transform 0.3s, opacity 0.3s;
}
.card-bom:hover {
  transform: scale(1.05) translateY(-4px);
  opacity: 0.9;
}
```

### Animações com View Timeline (Chrome 115+)

```css
@keyframes fade-in {
  from { opacity: 0; transform: translateY(20px); }
  to   { opacity: 1; transform: translateY(0); }
}

.card {
  animation: fade-in 0.6s ease-out both;
  animation-timeline: view();
  animation-range: entry 0% entry 50%;
}
```

## Scroll Suave com `scroll-behavior`

```css
html {
  scroll-behavior: smooth;
  scroll-padding-top: 80px; /* compensa header fixo */
}
```

## Conclusão

CSS moderno elimina a necessidade de bibliotecas JS para layout, animação e responsividade. Grid, Container Queries, `:has()` e `@layer` formam um conjunto robusto que cobre a maioria dos casos de UI.
