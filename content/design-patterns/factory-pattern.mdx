## Factory Pattern

O Factory Pattern é um padrão criacional que fornece uma interface para criar objetos em uma superclasse, mas permite que subclasses alterem o tipo de objetos que serão criados.

## Factory Method

Define um método para criar objetos, deixando as subclasses decidirem qual classe instanciar.

### Problema

Uma aplicação de logs precisa suportar diferentes destinos: arquivo, banco de dados, console. Em vez de instanciar diretamente cada tipo, usamos uma fábrica.

### Exemplo

```typescript
// Product interface
interface Logger {
  log(message: string): void;
}

// Concrete products
class FileLogger implements Logger {
  log(message: string) {
    console.log(`Escrevendo no arquivo: ${message}`);
  }
}

class DatabaseLogger implements Logger {
  log(message: string) {
    console.log(`Salvando no banco: ${message}`);
  }
}

class ConsoleLogger implements Logger {
  log(message: string) {
    console.log(`Console: ${message}`);
  }
}

// Creator
abstract class LoggerFactory {
  abstract createLogger(): Logger;

  logMessage(message: string) {
    const logger = this.createLogger();
    logger.log(message);
  }
}

// Concrete creators
class FileLoggerFactory extends LoggerFactory {
  createLogger(): Logger {
    return new FileLogger();
  }
}

class DatabaseLoggerFactory extends LoggerFactory {
  createLogger(): Logger {
    return new DatabaseLogger();
  }
}
```

## Abstract Factory

Fornece uma interface para criar famílias de objetos relacionados sem especificar suas classes concretas.

### Exemplo

```typescript
// Abstract products
interface Button {
  render(): void;
}

interface Input {
  render(): void;
}

// Concrete products para Windows
class WinButton implements Button {
  render() { console.log("Botão Windows"); }
}

class WinInput implements Input {
  render() { console.log("Input Windows"); }
}

// Concrete products para macOS
class MacButton implements Button {
  render() { console.log("Botão macOS"); }
}

class MacInput implements Input {
  render() { console.log("Input macOS"); }
}

// Abstract factory
interface UIFactory {
  createButton(): Button;
  createInput(): Input;
}

class WinFactory implements UIFactory {
  createButton(): Button { return new WinButton(); }
  createInput(): Input { return new WinInput(); }
}

class MacFactory implements UIFactory {
  createButton(): Button { return new MacButton(); }
  createInput(): Input { return new MacInput(); }
}
```

## Benefícios

- **Desacoplamento**: o cliente não conhece as classes concretas
- **Consistência**: garante que objetos de uma mesma família sejam usados juntos
- **Testabilidade**: facilita substituição de implementações em testes

## Quando Usar

- Quando a criação de objetos é complexa ou condicional
- Para isolar o código de criação da lógica de negócio
- Quando diferentes famílias de produtos precisam ser intercambiáveis

## Conclusão

Os padrões Factory são essenciais para escrever código desacoplado e aderente ao Princípio da Inversão de Dependências. Eles são amplamente utilizados em frameworks e bibliotecas modernas.
